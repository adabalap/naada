Naada v26 — install / upgrade
=============================

Frontend only this release. app.py, naada.sh and the deploy files are
unchanged from your repo.


UPGRADE
-------

  cd ~
  cp -r naada naada.backup          # rollback copy
  cd ~/naada
  ./naada.sh stop

  # unzip this archive so its naada/ folder replaces ~/naada
  # (or just copy static/ over — that's all that changed)

  ./naada.sh start

Two files worth keeping from your existing install:

  config.json      Not in this archive at all. Your Gemini key stays put.
  ai_cache.json    Regenerable cache. Keeping yours avoids re-fetching.

Your listening data — liked songs, playlists, folders, history, pinned
songs, settings — lives in the browser on your device, not in these files,
and survives the upgrade. For extra safety use
Home -> gear -> About Naada -> "Back up my data" first.


IMPORTANT: INSTALL TO YOUR HOME SCREEN
--------------------------------------

Open About Naada first. It now runs four live checks and tells you exactly
which one is failing, and shows an "Install Naada" button when Chrome
considers the app installable.

If you are opening Naada as a Chrome tab, two things will be wrong no
matter what the code does:

  * Music stops when you minimise. Android freezes background browser
    tabs far more aggressively than installed apps.
  * The media notification shows Chrome's icon, not Naada's. A web page
    cannot override that badge; only an installed app can.

Fix:  Chrome menu (three dots) -> Add to Home Screen
      Then always open Naada from that new icon.

To check which mode you are in, open About Naada. It now tells you
directly: green "Installed as an app", or amber "Running in a browser tab".

Samsung phones: also check Settings -> Battery -> Background usage limits,
and make sure Naada is not listed under "Sleeping apps".


FRESH INSTALL
-------------

  cd ~
  unzip naada-v26.zip
  cd naada
  pip install flask requests cryptography flask-cors
  ./naada.sh start

Chrome -> https://localhost:5443 -> accept the certificate ->
menu -> Add to Home Screen.

Optional, for AI features (story cards, trivia, mood playlists):

  1. Free key from https://aistudio.google.com
  2. cp config.example.json config.json
  3. Put your key in config.json
  4. ./naada.sh restart

Without a key the AI features hide themselves; everything else works.


AFTER STARTING
--------------

If the app looks unchanged it is serving a cached copy. Tap
Home -> gear -> About Naada -> "Update now", or hard-reload. The service
worker version was bumped, so this normally sorts itself out.


WHAT CHANGED
------------

See RELEASE-v26.md. Headlines: folders for grouping albums with
drag-to-file, the Chrome-icon and background-playback diagnosis, album
card layout hardening, and a settings entry point in Library so the
Light/Auto/Dark setting is actually reachable.
