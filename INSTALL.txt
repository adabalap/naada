Naada v33 — install / upgrade
=============================

This release changes static/ (frontend), and app.py gains the radio
endpoints. naada.sh keeps the "start-http" command from v27.

New in v33: PODCASTS (iTunes search + RSS, no API key), plus everything
from v28 -- internet radio, resume-where-you-left-off, sleep timer, recent
searches.

No new Python dependencies. The podcast feed parser uses the standard
library (xml.etree), and the proxies use "requests", which you already
have. Nothing to pip install.

Note: Trivia moved out of the bottom nav to make room for Podcasts. It is
now a row at the top of Library. Nothing was removed.


UPGRADE
-------

  cd ~
  cp -r naada naada.backup          # rollback copy
  cd ~/naada
  ./naada.sh stop

  # unzip this archive so its naada/ folder replaces ~/naada
  # (static/ and naada.sh both changed this time)

  ./naada.sh start-http      # <- see the next section for why

Two files worth keeping from your existing install:

  config.json      Not in this archive at all. Your Gemini key stays put.
  ai_cache.json    Regenerable cache. Keeping yours avoids re-fetching.

Your listening data — liked songs, playlists, folders, history, pinned
songs, settings — lives in the browser on your device, not in these files,
and survives the upgrade. For extra safety use
Home -> gear -> About Naada -> "Back up my data" first.


RUNNING BEHIND CLOUDFLARE (recommended)
---------------------------------------

Cloudflare gives you a real HTTPS certificate, which is what makes the PWA
installable -- and installing is what fixes both the media-notification
icon and background playback.

  ./naada.sh stop
  ./naada.sh start-cloud

start-cloud serves plain HTTP and trusts X-Forwarded-* headers. Do NOT let
Flask also do TLS behind Cloudflare; two TLS layers only produces handshake
errors.

Then open your https:// domain and use About Naada -> "Install Naada".

YOUR APP IS NOW PUBLIC. Anyone with the URL can use it and spend your
Gemini quota. To lock it down:

  export NAADA_TOKEN=pick-something-long
  ./naada.sh restart

Then visit  https://your-domain/?k=pick-something-long  once. A cookie is
stored for a year. Leave NAADA_TOKEN unset to keep it open.


RUNNING ON THE PHONE ONLY (no Cloudflare)
-----------------------------------------

  ./naada.sh stop
  ./naada.sh start-http

Then open http://localhost:5000. Plain http://localhost is trusted by
Chrome automatically, so it installs -- whereas https:// with a self-signed
certificate you clicked past can never be installed.

Open About Naada to check status: it runs four live checks and names the
failing one.

Samsung: check Settings -> Battery -> Background usage limits, and make
sure neither Chrome nor Termux is under "Sleeping apps".


FRESH INSTALL
-------------

  cd ~
  unzip naada-v33.zip
  cd naada
  pip install flask requests cryptography flask-cors
  ./naada.sh start-http

Chrome -> http://localhost:5000 -> About Naada -> Install Naada.

Optional, for AI features (story cards, trivia, mood playlists):

  1. Free key from https://aistudio.google.com
  2. cp config.example.json config.json
  3. Put your key in config.json
  4. ./naada.sh restart

Without a key the AI features hide themselves; everything else works.


AFTER STARTING
--------------

If the app looks unchanged it is serving a cached copy. Tap
Home -> gear -> About Naada -> "Update now", or hard-reload. The service
worker version was bumped, so this normally sorts itself out.
