Naada v27 — install / upgrade
=============================

This release changes static/ (frontend + icons + manifest) and adds a
"start-http" command to naada.sh. app.py is unchanged.


UPGRADE
-------

  cd ~
  cp -r naada naada.backup          # rollback copy
  cd ~/naada
  ./naada.sh stop

  # unzip this archive so its naada/ folder replaces ~/naada
  # (static/ and naada.sh both changed this time)

  ./naada.sh start-http      # <- see the next section for why

Two files worth keeping from your existing install:

  config.json      Not in this archive at all. Your Gemini key stays put.
  ai_cache.json    Regenerable cache. Keeping yours avoids re-fetching.

Your listening data — liked songs, playlists, folders, history, pinned
songs, settings — lives in the browser on your device, not in these files,
and survives the upgrade. For extra safety use
Home -> gear -> About Naada -> "Back up my data" first.


IMPORTANT: RUN IN HTTP MODE SO IT CAN BE INSTALLED
--------------------------------------------------

If you are opening Naada at https://localhost:5443 and clicking past a
certificate warning, Chrome will NEVER let you install it. A clicked-through
self-signed certificate leaves the origin flagged with a certificate error,
and Chrome refuses to install apps from such an origin.

Plain http://localhost, by contrast, is on Chrome's trusted-origins list and
needs no certificate at all. So HTTP is MORE installable than your HTTPS.

  ./naada.sh stop
  ./naada.sh start-http

Then open  http://localhost:5000  on the phone, and use the "Install Naada"
button in About Naada (or Chrome menu -> Add to Home Screen).

Installing is what fixes BOTH of these:
  * Music keeps playing when you minimise the app.
  * The media notification shows the Naada icon, not Chrome's.

start-http binds to 127.0.0.1 only. That is deliberate: http://<LAN-IP>:5000
is NOT a trusted origin, only literal "localhost" is. To reach Naada from
other devices, use the deploy/naada.adabala.com nginx config with your real
CA-signed certificate -- that installs fine too.

Open About Naada to check your status. It runs four live checks and tells
you exactly which one is failing.

Samsung: also check Settings -> Battery -> Background usage limits and make
sure neither Chrome nor Termux is under "Sleeping apps".


FRESH INSTALL
-------------

  cd ~
  unzip naada-v27.zip
  cd naada
  pip install flask requests cryptography flask-cors
  ./naada.sh start-http

Chrome -> http://localhost:5000 -> About Naada -> Install Naada.

Optional, for AI features (story cards, trivia, mood playlists):

  1. Free key from https://aistudio.google.com
  2. cp config.example.json config.json
  3. Put your key in config.json
  4. ./naada.sh restart

Without a key the AI features hide themselves; everything else works.


AFTER STARTING
--------------

If the app looks unchanged it is serving a cached copy. Tap
Home -> gear -> About Naada -> "Update now", or hard-reload. The service
worker version was bumped, so this normally sorts itself out.


WHAT CHANGED
------------

See RELEASE-v27.md. Headlines: folders for grouping albums with
drag-to-file, the Chrome-icon and background-playback diagnosis, album
card layout hardening, and a settings entry point in Library so the
Light/Auto/Dark setting is actually reachable.
